# 스프링 프로젝트 생성

- 이전의 **DDD(Domain Driven Design) 디렉토리 구조**를 채택.
  - 협업을 할 때 대부분 도메인 별로 나눠서 작업하기 때문에, 도메인별로 디렉토리를 나누니까 효율적으로 작업할 수 있었음.
  - 또한, 도메인별로 나누니까 각 파일들의 위치를 찾는 것도 더 쉬웠고 효율적이었음.

- 풀코스 백엔드를 회고/정리하면서 스프링 프로젝트 생성함.
